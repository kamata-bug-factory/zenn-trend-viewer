# Zenn Trend Viewer 仕様書

## 概要

Zennのトレンド記事をブラウザのツールバーからワンクリックで確認できるChrome拡張機能。

---

## 機能一覧

### 1. トレンド記事一覧表示

ツールバーのアイコンをクリックするとポップアップが開き、Zennのトレンド記事を一覧表示する。

**表示項目（1記事あたり）:**

| 表示順 | 項目         | 取得元   | 表示形式                                   |
| ------ | ------------ | -------- | ------------------------------------------ |
| 1      | 記事タイトル | Zenn API | テキスト                                   |
| 2      | 著者名       | Zenn API | テキスト                                   |
| 3      | 投稿日時     | Zenn API | 相対時刻（「〇分前」「〇時間前」など）     |
| 4      | LGTM数       | Zenn API | 数値（`liked_count`）                      |
| 5      | タグ         | Zenn API | タグ一覧（`topics`）                       |

**表示件数:** Tech / Idea それぞれ上位10件（[qiita-trend-viewer](https://github.com/kamata-bug-factory/qiita-trend-viewer) と件数を揃える）

### 2. Tech / Idea 切り替えトグル

ヘッダーに Tech / Idea を切り替えるトグルを配置し、選択されたカテゴリのトレンド記事一覧を表示する。

**トグルの仕様:**

| 項目         | 仕様                                       |
| ------------ | ------------------------------------------ |
| 配置         | ポップアップ上部のヘッダー                 |
| 選択肢       | Tech / Idea                                |
| 初期表示     | Tech                                       |
| 切替時の挙動 | 該当カテゴリのキャッシュまたはAPIから表示 |

---

## データ取得

### トレンド記事の取得

Zenn API を直接呼び出してトレンド記事一覧を取得する。Tech と Idea で別エンドポイントを使用する。

| カテゴリ | エンドポイント                                                |
| -------- | ------------------------------------------------------------- |
| Tech     | `https://zenn.dev/api/articles?order=daily&article_type=tech` |
| Idea     | `https://zenn.dev/api/articles?order=daily&article_type=idea` |

| 項目         | 仕様                                                                |
| ------------ | ------------------------------------------------------------------- |
| 認証         | 不要                                                                |
| 取得件数     | レスポンスの上位10件を使用                                          |
| 取得データ   | 記事タイトル、著者名、投稿日時、LGTM数、タグ一覧、記事URL           |

> 1回のAPI呼び出しでタグ・LGTM数まで含む記事情報が取得できるため、qiita-trend-viewer のような N+1 リクエスト対策は不要。

---

## キャッシュ

| 項目     | 仕様                                                  |
| -------- | ----------------------------------------------------- |
| 保存先   | `chrome.storage.local`                                |
| 有効期限 | 3時間                                                 |
| 対象     | 取得済みの記事情報（Tech / Idea を別キーで保持）      |

**キャッシュフロー:**

1. ポップアップ表示時、選択中カテゴリ（Tech / Idea）のキャッシュを確認
2. キャッシュが存在し有効期限内 → キャッシュから表示
3. キャッシュが存在しないまたは期限切れ → Zenn API から再取得し、キャッシュを更新
4. トグル切替時も同様にキャッシュを優先して表示

---

## 技術構成

| 項目       | 技術                                             |
| ---------- | ------------------------------------------------ |
| 形式       | Chrome拡張機能（Manifest V3）                    |
| UI         | ポップアップ（ツールバーアイコンクリックで表示） |
| ストレージ | `chrome.storage.local`                           |
| 外部通信   | Fetch API                                        |

UI構成・スタイリング方針（React + shadcn/ui + Tailwind CSS）は qiita-trend-viewer と揃える。
