# Zenn Trend Viewer 仕様書

## 概要

Zennのトレンド記事をブラウザのツールバーからワンクリックで確認できるChrome拡張機能。

---

## 機能一覧

### 1. トレンド記事一覧表示

ツールバーのアイコンをクリックするとポップアップが開き、Zennのトレンド記事を一覧表示する。

**表示項目（1記事あたり）:**

| 表示順 | 項目         | 取得元            | 表示形式                                   |
| ------ | ------------ | ----------------- | ------------------------------------------ |
| 1      | 記事タイトル | Zenn API（一覧）  | テキスト                                   |
| 2      | 著者名       | Zenn API（一覧）  | テキスト                                   |
| 3      | 投稿日時     | Zenn API（一覧）  | 相対時刻（「〇分前」「〇時間前」など）     |
| 4      | LGTM数       | Zenn API（一覧）  | 数値（`liked_count`）                      |
| 5      | タグ         | Zenn API（詳細）  | タグ一覧（`topics`）                       |

**表示件数:** Tech / Idea それぞれ上位10件（[qiita-trend-viewer](https://github.com/kamata-bug-factory/qiita-trend-viewer) と件数を揃える）

### 2. Tech / Idea 切り替えトグル

ヘッダーに Tech / Idea を切り替えるトグルを配置し、選択されたカテゴリのトレンド記事一覧を表示する。

**トグルの仕様:**

| 項目         | 仕様                                       |
| ------------ | ------------------------------------------ |
| 配置         | ポップアップ上部のヘッダー                 |
| 選択肢       | Tech / Idea                                |
| 初期表示     | Tech                                       |
| 切替時の挙動 | 該当カテゴリのキャッシュまたはAPIから表示 |

---

## データ取得

Zenn API を2段階で呼び出してトレンド記事を取得する。
タイトル・著者・投稿日時・LGTM数は一覧エンドポイントから、タグ（`topics`）は記事ごとの詳細エンドポイントから取得する。

### 1. 一覧取得

Tech と Idea で別エンドポイントを使用する。

| カテゴリ | エンドポイント                                                |
| -------- | ------------------------------------------------------------- |
| Tech     | `https://zenn.dev/api/articles?order=daily&article_type=tech` |
| Idea     | `https://zenn.dev/api/articles?order=daily&article_type=idea` |

| 項目       | 仕様                                                              |
| ---------- | ----------------------------------------------------------------- |
| 認証       | 不要                                                              |
| 取得件数   | レスポンスの上位10件を使用                                        |
| 取得データ | 記事ID、slug、記事タイトル、著者名、投稿日時、LGTM数、記事URL     |

### 2. 詳細取得（タグ補完）

一覧エンドポイントのレスポンスには `topics` が含まれないため、タグ表示のために slug 単位で詳細APIを呼び出す。

| 項目           | 仕様                                             |
| -------------- | ------------------------------------------------ |
| エンドポイント | `https://zenn.dev/api/articles/:slug`            |
| 認証           | 不要                                             |
| 呼び出し回数   | 表示対象の上位10件それぞれに対して呼び出し       |
| 実行方法       | `Promise.all` で並列実行                         |
| 取得データ     | タグ一覧（`topics`）                             |

> 上位10件に絞ってから `Promise.all` で並列実行することで、N+1 リクエストの影響を抑える。

## キャッシュ

| 項目     | 仕様                                                  |
| -------- | ----------------------------------------------------- |
| 保存先   | `chrome.storage.local`                                |
| 有効期限 | 3時間                                                 |
| 対象     | 取得済みの記事情報（Tech / Idea を別キーで保持）      |

**キャッシュフロー:**

1. ポップアップ表示時、選択中カテゴリ（Tech / Idea）のキャッシュを確認
2. キャッシュが存在し有効期限内 → キャッシュから表示
3. キャッシュが存在しないまたは期限切れ → Zenn API（一覧 + 詳細）から再取得し、キャッシュを更新
4. トグル切替時も同様にキャッシュを優先して表示

## 技術構成

| 項目       | 技術                                             |
| ---------- | ------------------------------------------------ |
| 形式       | Chrome拡張機能（Manifest V3）                    |
| UI         | ポップアップ（ツールバーアイコンクリックで表示） |
| ストレージ | `chrome.storage.local`                           |
| 外部通信   | Fetch API                                        |

UI構成・スタイリング方針（React + shadcn/ui + Tailwind CSS）は qiita-trend-viewer と揃える。
